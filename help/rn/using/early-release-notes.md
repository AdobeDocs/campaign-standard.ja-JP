**強化点**

* Microsoft Dynamics 365の統合は、専用のセルフサービス統合アプリと改善された実装プロセスによって強化されました。 [詳細情報](../../integrating/using/d365-acs-get-started.md)

* 特定のプロセスが原因で、配信の実行が非常に遅くなる問題を修正しました。 これは、いくつかのパラメーターに対して誤った単位が定義されていたためです（例：秒ではなくミリ秒）。

* deliveryId/MessageIDがnullでないという条件に基づいて、モバイルSDKが開いているトラッキングリクエストを送信した場合の問題を修正しました。 これにより、追跡が無効になっている配信で404エラーが発生します。 配信の追跡ステータスに関する情報を含む追加の変数(acsDeliveryTracking)がペイロードで送信されるようになりました。 この変数には、トラッキングステータスの設定に応じて、2つの値をonまたはoffに設定できます。

* トランザクションメッセージングプロセスで問題が発生した場合のトラブルシューティングを容易にするための改善が行われました。 Adobeの技術管理者は、任意のプロセスでトレースを再起動せずに使用できるようになりました。

* プロファイルリストで、次のいずれかのフィールドに基づいてレコードを検索できるようになりました。プロファイルリソースを拡張する際のアドバンスフィルターで追加された電子メール、名、姓、またはカスタムフィールド。 この機能は、filterTypeパラメーターを使用するCampaign StandardAPIでも利用できます。

* トランザクションメッセージングデータベースプール処理を実行しているコンテナの数に対してパラメーターが調整されました。 これにより、使用されるすべてのコンテナに負荷を均等に分散させ、最適なパフォーマンスを得ることができます。

* 外部パラメーターを使用してワークフローを呼び出した後、イベント変数を使用するアクティビティで新しい関数(GetOption)が使用できるようになりました。 指定した関数の値を返すことができます。

* 新しい技術的なオプションが追加されました。 ワークフローを開始する前に、Campaign Standardがシステムに十分な物理メモリがあるかどうかを確認できます。 メモリ量が少なすぎる場合は、システムメモリがこのしきい値に達するまでワークフローの実行が遅れます。 これは、パフォーマンスのさらなる低下を回避し、停止のリスクを軽減するために行われます。 アクティビティが少ない時間にこのワークフローを再スケジュールしてから、やり直してください。 サーバーの負荷が緩和されると、ワークフローは自動的に再開されます。 このオプションは読み取り専用で、変更できません。

**その他の変更**

* メッセージの準備中に、経時変化1時間あたりのコンテンツダウンロード数が100件の制限に達した場合の警告に変更されました。 上限に達すると警告が表示され、配信を続行できます。

* トランザクションプッシュメッセージのターゲットプロファイルで、新しい配信マッピング（mapRtEventAppSubRcp）を使用できるようになりました。これらの配信の配信ログ、除外ログおよびトラッキングログは、broadLogAppSubRcp、excludeLogAppSubRcp、trackingLogAppSubRcp の各テーブルで使用できるようになりました。これにより、プロファイルターゲットディメンションを使用してトランザクションプッシュメッセージを送信する際に、配信分析が失敗する問題が解決されました。

* トランザクションメッセージコンテンツを富化する際に、プロファイルテーブルからデータを取得する際に、リンクが検索されなくなり、メッセージ準備中の待ち時間が減少し、プロファイルテーブルに定義された誤った関係に起因する空のプロファイルデータが回避されます。

* インスタンスの技術的な設定は、安定性を確保するために最適化されました。 （CAMP-45681）

* セッション管理が改善され、メモリが最適化されました。 （CAMP-45901、CAMP-46767）

* **転送ファイル**&#x200B;アクティビティは、アップロードまたはダウンロードされたファイルの数を含む追加の変数(filesCount)を生成するようになりました。 （CAMP-45842）

* SMSコネクタは、各メッセージに複数のオプションパラメータを送信できるようになりました。

* DATA MODELロールを持つ配信がユーザーログ拡張子を発行できない問題を修正しました。 この操作は、DATA MODELロールで使用できるようになります。 （CAMP-46604）

* 1回実行された&#x200B;**重複排除 - 重複**&#x200B;アクティビティをコピー&amp;ペーストし、一時的なリソースを利用する場合に発生するワークフローの問題を修正しました。 複製が行われると、アクティビティのリソースは自動的に空に設定され、ワークフローの他のアクティビティで問題が発生します。 貼り付けたアクティビティのリソースは、可能な限り早くワークフロー内で後ではなく、同じままになります。 （CAMP-46903）

* 存在しなくなったカスタムリソースをターゲットにしたリソースを発行しようとすると表示されるエラーメッセージが明確になりました。 （CAMP-46893）

* 次の言語が優先言語リストに追加されました。インドネシア語 — インドネシア語(in-id)、英語 — スウェーデン語(en-se)、英語 — アジア太平洋語(en-ap)、英語 — 日本(en-jp)、スペイン語 — ラテンアメリカ(es-la)。 （CAMP-46351）

* ランディングページのテスト時にプロファイルを選択するピッカーが、タイムアウトを防ぐために、プロファイルの代わりにprofilBaseリソースを使用するようになりました。

* SMPPログ形式が改善されました。

* cryptStringおよびdecryptString JS関数に対するオプションのパラメーターが、Adobe Campaign ClassicAPIに合わせて追加されました。

* 配信の準備ログの警告メッセージまたはエラーメッセージを改善しました。

* IMSに接続しようとした場合のエラーログを改善。

* 動的レポートの検索バーを使用して、配信ディメンションとキャンペーンディメンションをさらにフィルターできるようになりました。

* トランザクションSMSメッセージの有効期限を、トランザクションメッセージAPIの有効期限パラメータの値セットで定義できるようになりました。 （CAMP-36600）

* 動的レポートで、**配信サマリ**&#x200B;組み込みレポートに、登録解除率指標に対して正しくないデータが表示されていました。 この問題を修正するために、**実購読解除**&#x200B;という名前の新しい指標が追加されました。 （CAMP-46445）
