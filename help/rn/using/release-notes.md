# リリースノート{#release-notes}

Adobe Campaign Standardの特定のリリースをお探しですか？

各リリースには、新しい機能とパッチが付属しています。 リリースをクリックして、その内容を表示します。 次のリリース [がいつ発生するかを確認するには](https://helpx.adobe.com/campaign/kb/acs-release-planning.html) 、「リリース計画」を参照してください。

Adobe Campaign Standardの最新のドキ [ュメントの更新](../../rn/using/documentation-updates.md) . 以前のリリースをお探しの場合は、次のページを参照してください。 [2018 Release Notes](../../rn/using/release-notes-2018.md), [2017 Release Notes](../../rn/using/release-notes-2017.md), [2015-2016 Release Notes](../../rn/using/release-notes-2015-2016.md). また、非推奨機能と削除された機 [能のリストも参照してください](https://helpx.adobe.com/campaign/kb/acs-deprecated-and-removed-features.html)。

## コントロールパネルの更新- 2019年8月 {#controlpanel-update---august-2019}

### 何が新しいの？ {#what-s-new-4}

ドメインのSSL証明書の有効期限が切れる前に、Adminユーザーが通知を受け取る新しい機能が追加されました。  詳細については、詳細なドキュメントを参照 [してください](https://helpx.adobe.com/campaign/kb/control-panel-subdomains-certificates.html)。

さらに、管理者ユーザーは、SFTPサーバーにアクセスするために追加されたSSHキーを削除できるようになりました。

Control Panelは、AWSでホストされるお客様に対してのみ利用できます。 これらのアップデートは8月26日にご利用いただけます。

### その他の改善点 {#other-improvements-3}

* Adobe Experience Platform Location Servicesとの統合により、Adobe Campaignは、Experience Platform SDKを使用して、場所ベースのマーケティングメッセージをモバイルアプリケーションの購読者に送信できるようになりました。  詳細については、詳細なドキュメントを参照 [してください](../../integrating/using/configuring-campaign-points-of-interest-data-integration.md)。
* レポート機能が向上し、より良い経験が得られました。 この機能を使用するには、Dynamic Reporting使用許諾契約に同意する必要があります。 詳細については、詳細なドキュメントを参照 [してください](../../reporting/using/about-dynamic-reports.md#dynamic-reporting-usage-agreement)。
* ワークフローでは、ワークフローの次の10回の実行をプレビューする新しいオプションが追加されました。 詳細については、詳細なドキュメントを参照 [してください](../../automating/using/scheduler.md)。
* スケジューラ·アクティビティでは、新しいオプションを使用して、月次搬送に対して特定の曜日を選択できます。 詳細については、詳細なドキュメントを参照 [してください](../../automating/using/scheduler.md)。
* 集計期間のない定型搬送を作成する場合、搬送ダッシュボードでは、搬送が送信される前に確認を要求できるようになりました。 詳細については、詳細なドキュメントを参照 [してください](../../sending/using/confirming-the-send.md)。
* ワークフローの外部シグナルアクティビティで宣言されたイベント変数を使用して、配信のラベルをパーソナライズできるようになりました。 詳細については、詳細なドキュメントを参照 [してください](../../automating/using/calling-a-workflow-with-external-parameters.md)。
* GDPR削除クエリは、パフォーマンスを向上させるために改良されました。 (CAMP-33504)
* 外部アカウント構成インターフェイスから"ftp"オプションが削除されました。 (CAMP-34472)
* 各電子メールメッセージに対して、SMTPテストモードオプションを有効または無効にできます。 詳細については、詳細なドキュメントを参照 [してください](../../administration/using/configuring-email-channel.md#smtp-test-mode)。 (CAMP-34602)

### その他の変更点 {#other-changes-2}

* 配信プロパティインターフェイスに警告が追加されました。 このメソッドは、集約期間に基づいて配信を準備し、ワークフローを1日に複数回呼び出すように解凍することを指定します。配信に期間がないことを確認する必要があります。 (CAMP-34393)
* カスタムリソース構成画面に警告が追加されました。 カスタムリソースIDには、最大30文字を使用することをお勧めします。 これは、ユーザー設定のリソースフィールド、キー、インデックス、およびリンクにも適用されます。
* ランディング·ページで確認メッセージとして使用されるトランザクション·メッセージを削除しようとすると、メッセージが表示されるようになりました。
* アクティビティが6時間以上実行されている場合、ワークフローログに警告が表示されるようになりました。 これは、プッシュ通知、配信、シグナル、開始、終了、フォーク、ANDジョイント、スケジュール、および待機の各アクティビティには適用されません。
* 同時に実行されているワークフローの最大数に達すると、ワークフローログに警告が表示されるようになりました。
* 7日を超える一時停止または失敗状態のワークフローは、ディスク領域の消費を減らすために停止されました。 クリーニングタスクがワークフローログに表示されます。
* 「ファイルの転送」アクティビティを使用する場合、ファイルサイズが使用可能なディスク領域を超えると、エラーがログに記録されます。
* In-Appメッセージのセカンダリボタンに対して、[宛先URLにリダイレクト]操作を選択できなくなりました。
